# 流程流转图功能实现 - 演示指南

## 功能概述

将模版详情从简单列表改造为**可交互的流程流转图**，支持：

✅ **可视化流程** - 清晰的阶段连接图示  
✅ **来源标记** - 🔒继承、✂️禁用、➕新增  
✅ **快速预览** - 每个阶段的任务列表  
✅ **可拖拽编辑** - 母版编辑时支持重新排序（后续功能）

## 流程流转图特性

### 1. 阶段卡片设计

每个卡片包含：
- **圆形序号** - 蓝色背景，显示阶段顺序
- **阶段名称** - 粗体，易于识别
- **来源标记** - 🔒继承、✂️禁用、➕新增
- **描述文字** - 灰色小字，说明阶段内容
- **任务数量** - 📋显示包含的任务数
- **禁用状态** - 禁用阶段显示灰色背景和禁用徽章

### 2. 连接线

- **SVG 绘制** - 平滑的竖向箭头连接
- **自动生成** - 在每个卡片下方自动显示
- **最后卡片无连接** - 流程终点不显示下一步箭头

### 3. 交互体验

```
点击卡片 → 可查看该阶段的详细任务列表
         → 显示任务状态、优先级、负责人

悬停效果 → 卡片边框变为蓝色
         → 卡片微微上移
         → 显示阴影效果
```

### 核心代码

```jsx
import ProcessFlowDiagram from '../components/ProcessFlowDiagram';

// 使用示例
<ProcessFlowDiagram
  phases={phases}
  editable={false}  // 预览模式
  onPhaseSelect={(phase) => console.log(phase)}  // 点击事件
/>

// 编辑模式
<ProcessFlowDiagram
  phases={phases}
  editable={true}  // 启用拖拽
  onPhasesChange={(newPhases) => saveChanges(newPhases)}  // 保存排序

```

### 关键特性

#### 1. 来源标记
```javascript
const getSourceIcon = (phase) => {
  if (phase.disabled) return '✂️ 禁用';      // 禁用的阶段
  if (phase.source === 'new') return '➕ 新增';   // 新增阶段
  return '🔒 继承';                              // 继承的阶段
};
```

#### 2. 拖拽排序（编辑模式）
- 使用 `dnd-kit` 库实现轻量级拖拽
- 自动重新计算 `order` 字段
- 变更保存到数据库

#### 3. 流程连接
```javascript
// 最后一个阶段不显示连接线
{phase.order < 12 && (
  <div className="phase-connector">
    {/* SVG 连接线和箭头 */}
  </div>
)}
```

## 使用流程

### 用户视角

1. **打开模版库** → 导航到 `/template-library`
2. **选择模版** → 点击左侧模版列表
3. **查看流程** → 右侧显示流程流转图
4. **点击阶段** → 可查看该阶段详细信息
5. **应用模版** → 点击下方"应用模版"按钮

### 管理员视角

1. **编辑模版** → 打开模版编辑页面（待实现）
2. **拖拽调整** → 拖动阶段卡片重新排序
3. **禁用阶段** → 点击阶段卡片禁用选项
4. **保存** → 自动保存排序和禁用状态

| 标记 | 类型 | 说明 |
|------|------|------|
| 🔒 | 继承 | 来自父模版，未修改 |
| ✂️ | 禁用 | 来自父模版，本子模版不需要 |
| ➕ | 新增 | 本子模版独有的新增阶段 |

## 故障排除

### 问题：流程图不显示
**解决**：检查 `ProcessFlowDiagram` 组件是否正确导入
```jsx
import ProcessFlowDiagram from '../components/ProcessFlowDiagram';
```

### 问题：拖拽不工作
**解决**：确保 `editable={true}` 且已安装 `@dnd-kit` 库
```bash
npm list @dnd-kit/core
```

### 问题：样式显示错误
**解决**：检查 CSS-in-JS 支持，确保使用 Styled JSX
```jsx
<style jsx>{`...`}</style>
```
