# 研发项目管理系统 - 需求文档 v3.0

> 基于《研发部项目3月进展.pdf》实际场景定制

---

## 1. 产品定位

**名称**：研发项目管理系统（R&D Project Management System）

**定位**：面向研发部门的项目进度跟踪与协作平台，支持多类型项目管理（平台研发、定制开发、合作研发、性能测试、应用推广），实现项目全生命周期可视化跟踪。

**目标用户**：
- 研发主管/经理（全局视图）
- 项目经理（项目跟踪）
- 研发工程师（任务执行）
- 部门负责人（汇报决策）

## 3. 项目分类体系（基于PDF内容）

### 3.1 项目类型（一级分类）
| 类型 | 子类 | 说明 |
|------|------|------|
| **平台项目** | 2.0C平台、2.0B平台、3.0平台 | 核心产品开发 |
| **定制项目** | 科研定制、芯片定制、玻璃定制 | 客户定制需求 |
| **合作项目** | 海南大学、黑马、贝索 | 外部合作研发 |
| **测试项目** | MDX131、引探测试、性能验证 | 内部测试验证 |
| **应用项目** | 水产、食品安全、教育版 | 场景应用推广 |

### 3.2 项目状态
| 状态 | 说明 |
|------|------|
| 规划中 | 需求确认阶段 |
| 进行中 | 设计/开发/测试中 |
| 待加工 | 图纸外发，等待加工件 |
| 待验证 | 加工完成，等待测试验证 |
| 已完成 | 交付完成 |
| 已归档 | 历史项目 |

### 3.3 项目阶段
- 需求确认 → 设计 → 外发加工 → 采购 → 组装 → 调试 → 验证 → 交付

## 8. 技术架构

### 8.1 前端技术栈
- **框架**: React 18 + TypeScript
- **构建**: Vite
- **样式**: Tailwind CSS
- **UI组件**: shadcn/ui
- **图表**: Recharts
- **路由**: React Router
- **状态管理**: React Query

### 8.2 后端技术栈
- **云函数**: Node.js CloudBase Functions
- **认证**: CloudBase Auth
- **数据库**: CloudBase 文档数据库
- **存储**: CloudBase 文件存储（报告导出）

### 8.3 部署方案
- **前端**: CloudBase 静态托管
- **后端**: CloudBase 云函数
- **域名**: 绑定自定义域名

## 9. 系统特色（基于PDF分析）

### 9.1 与现有流程的衔接
1. **项目定位**: 每个项目有明确的定位描述
2. **参与人员与实施详情**: 记录每个人的具体工作
3. **项目综合判断**: 对项目整体状态的评估
4. **时间轴管理**: 3月→4月→5月的连续跟踪

### 9.2 解决现有痛点
| 痛点 | 解决方案 |
|------|----------|
| 手工整理Excel/PDF耗时 | 系统自动生成报告 |
| 项目状态不透明 | 仪表盘实时展示 |
| 人员工作不清晰 | 个人工时统计 |
| 历史数据难追溯 | 完整历史记录 |
| 跨项目对比困难 | 统一视图对比 |
